# 后端开发规则

## 技术栈

- FastAPI 0.100+、MySQL、Redis、JWT、Python 3.9+

## 编码规范

- 遵循 PEP 8 规范
- 使用类型提示，函数和方法要有文档字符串
- **编程范式**: 优先使用函数式编程风格
  - 避免使用可变状态
  - 优先使用纯函数
  - 合理使用高阶函数和闭包
  - 减少副作用

## 日志管理

- 存储路径: `./logs/`
- 文件命名: `error.log`、`access.log`
- 日志格式: 包含时间、级别、IP、方法、路径、状态码、响应时间
- 轮转策略: 10MB 大小轮转，每天凌晨 0 点时间轮转，保留 7 天

## 项目结构

```
backend/
├── app/
│   ├── api/         # API 路由
│   ├── business/    # 业务模块（按功能划分）
│   │   ├── auth/    # 认证业务模块
│   │   ├── user/    # 用户业务模块
│   │   └── ...      # 其他业务模块
│   ├── utils/       # 工具函数
│   ├── core/        # 核心配置
│   └── main.py      # 应用入口
├── logs/            # 日志目录
├── requirements.txt # 依赖管理
└── .env             # 环境变量
```

## 验证标准

- PEP 8 规范: 使用 `flake8` 检查代码
- 日志功能: 验证日志生成、记录和轮转
- JWT 认证: 测试 token 获取、使用和过期处理

## 最佳实践

- 使用依赖注入管理数据库连接
- 实现统一的异常处理机制
- 使用 Pydantic V2 进行数据验证
- 定期备份 MySQL 数据
- 使用环境变量管理敏感配置
- 合理使用 Redis 进行缓存和会话管理
